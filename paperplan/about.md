# Project Planning Methodology

This project uses the same planning methodology defined in the Radio monorepo (`radio/paperplan/about.md`).

## Quick Start

1. Plans live in `paperplan/` directory
2. Named as `XXX-DESCRIPTIVE_NAME_PLAN.md`
3. Track progress with phases and checkboxes
4. Update `plans.md` with current status

## Plan Structure

Every plan follows this template:
- **Overview** — What the plan achieves
- **Goal** — One-sentence objective
- **Phases** — Sequential tasks with checkboxes
- **Key Decisions** — Documented rationale
- **Definition of Done** — Clear completion criteria
- **Notes** — Risks, dependencies, context

## Status Values

| Status | Meaning |
|--------|---------|
| ⏳ Pending | Not started |
| 🚧 In Progress | Currently working |
| ✅ Complete | Done and verified |
| ❌ Cancelled | Will not be done |
